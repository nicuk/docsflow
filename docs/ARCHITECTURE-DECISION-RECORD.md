## **🔧 TECHNICAL DECISIONS**

### **Authentication Architecture**
**Decision**: Use Supabase Auth with JWT tokens
**Rationale**: 
- Built-in user management
- Row Level Security (RLS) ready
- Social auth support
- Scales with multi-tenancy

**Implementation**:
```typescript
// Backend: lib/auth.ts
import { createClient } from '@supabase/supabase-js'

export const supabase = createClient(
  process.env.SUPABASE_URL!,
  process.env.SUPABASE_SERVICE_KEY!
)

// Frontend: lib/supabase-client.ts  
import { createBrowserClient } from '@supabase/ssr'

export const supabase = createBrowserClient(
  process.env.NEXT_PUBLIC_SUPABASE_URL!,
  process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!
)
```

### **Tenant & Persona Architecture**
**Decision**: Store personas in Supabase with LLM generation
**Rationale**:
- Personas are core differentiator
- Need persistent storage
- Must be tenant-specific

**Schema**:
```sql
-- Tenants table (already exists)
ALTER TABLE tenants ADD COLUMN IF NOT EXISTS
  custom_persona JSONB DEFAULT NULL;

-- Persona structure
{
  "role": "Expert Motorcycle Dealership Assistant",
  "tone": "Friendly and conversational",
  "focus_areas": ["inventory", "sales", "customer service"],
  "prompt_template": "Generated by LLM...",
  "industry": "motorcycle_dealer",
  "created_from": "onboarding_answers"
}
```
